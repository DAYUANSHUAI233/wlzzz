# 股票市场数据中转站 - 项目总览

## 项目简介

这是一个股票市场数据聚合中转站，从多个第三方数据源 API 获取数据，并统一对外提供 RESTful API 接口。项目已从本地 Flask 应用迁移到腾讯云无服务器函数（SCF），实现零运维、低成本、高可用的云端服务。

---

## 核心功能

### 1. 数据源聚合

本项目整合了多个数据源 API：

| 数据源 | 用途 | 接口类型 |
|--------|------|----------|
| longhuvip | 板块排名、成分股、量能数据 | HTTP GET/POST |
| huanshoulv | 市场情绪、涨停统计 | HTTP GET |
| wogoo | 大跌回撤、资金趋势 | HTTP POST |
| zizizaizai | 交易日历 | HTTP GET |

### 2. 主要接口

#### 基础接口

- **板块排名**: 获取实时题材板块热度排名
- **成分股查询**: 查询指定板块的成分股列表

#### 市场数据接口

- **上涨下跌家数**: 统计当日上涨和下跌股票数量及占比
- **大幅回撤与强度**: 统计大幅回撤数量和市场综合强度
- **量能数据**: 提供当前总量、昨日总量及预测量能
- **市场情绪核心**: 包含首板、二板、涨停、炸板率等关键指标
- **大跌回撤统计**: 统计主线大跌和大幅回撤数据
- **资金趋势**: 提供涨停排队资金、成交资金等资金流向数据
- **默认交易日**: 获取当前交易日日期

#### 聚合接口

- **/all**: 一次性获取所有市场数据 + 前 10 名热门题材及成分股

## 安全考虑

### API 密钥管理

- 当前代码中包含第三方 API 的请求头和 Token
- **建议**: 将敏感信息存储在云函数环境变量或腾讯云密钥管理服务（KMS）中

### 网络安全

- 使用 HTTPS 加密传输
- API 网关自带 DDoS 防护
- 建议配置访问白名单（如需要）

### 访问控制

- 可在 API 网关配置 API Key 或签名验证
- 代码中已配置 CORS，允许跨域访问

## 扩展建议

### 1. 添加缓存层

使用腾讯云 COS 或 Redis 缓存热点数据：

```python
# 伪代码示例
def get_cached_data(key, ttl=60):
    # 先查缓存
    cached = redis.get(key)
    if cached:
        return json.loads(cached)

    # 缓存未命中，请求 API
    data = fetch_api_data()

    # 写入缓存
    redis.setex(key, ttl, json.dumps(data))
    return data
```

**优势**: 减少 API 调用，降低延迟，节省成本

### 2. 配置 CDN 加速

为 API 网关配置 CDN：

- 加速静态资源响应
- 减少云函数调用次数（边缘缓存）
- 降低延迟

### 3. 添加 API 鉴权

在 API 网关配置鉴权：

- API Key 鉴权
- OAuth 2.0
- 自定义签名验证

### 4. 配置告警

在云监控配置告警规则：

- 函数错误率 > 5%
- 执行超时
- 调用次数异常

## 故障处理

### 常见故障

| 故障现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 函数超时 | 外部 API 响应慢 | 增加超时时间或优化代码 |
| 错误率上升 | 数据源 API 异常 | 检查日志，联系数据源 |
| 冷启动慢 | 函数未预热 | 使用预留并发实例 |
| 404 错误 | API 网关路由错误 | 检查触发器配置 |

### 应急预案

1. **降级策略**: 返回缓存数据或默认值
2. **重试机制**: 失败后自动重试
3. **熔断机制**: 连续失败后停止调用
4. **回滚方案**: 使用版本管理快速回滚
